December 11, 2007
Technical Overview of OSGi

This month's meeting will feature an overview of the Open Gateway Standards initiative (OSGi). Formed in 1999, OSGi was originally designed to promote dynamic systems for embedded Java and network devices. OSGi is a lightweight dynamic module system that helps you:
  • cope with "classpath hell"
  • load multiple versions of a class
There are a few implementations of OSGi, namely:
  • Equinox (Eclipse)
  • Apache Felix
  • Knopflerfish
  • Spring OSGi
We will take a look at how to build your first "bundle," i.e., module and look at how to use OSGi's services.
