September 10, 2002
A Practical Use of the Facade Design Pattern

This month's meeting will be a source code review of an application utilizing the Facade design pattern. Design patterns are "simple and elegant solutions to specific problems in object-oriented software design." 1 The Facade Pattern is used to wrap a complex set of classes into a much simpler "interface." A practical use of the Facade Pattern is to encompass the classes found within the JDBC. The application is based on the Facade Pattern chapter found in Java Design Patterns - A Tutorial written by James W. Cooper. There are two classes that make up the "interface" to all the classes within the java.sql package, and the remainder of the application presents the data from the database into a frame.

1 Erich Gamma...[et al.]. Design Patterns - Elements of Reusable Object-Oriented Software, 1995
