Abstract

Calling all GSJUG and NYJavaSIG members!

Java 21 is packed with exciting new features that will revolutionize how you write Java code. Immerse yourself in the mystical realm of virtual threads, where data manipulation becomes more concise and efficient than ever. Witness the magic of disappearing records, adding a touch of elegance and simplicity to your codebase. Traverse collections in reverse, opening up new possibilities in data manipulation and analysis.

Additionally, discover the revamped "Hello World" experience in Java 21, designed to welcome new developers with open arms and ease them into the Java ecosystem. Dabble in the art of structural concurrency, gaining mastery over task execution across multiple threads like never before.

Are you ready to embrace the power of Java 21 and embark on a remarkable journey of innovation? Join us at the joint GSJUG/NYJavaSIG September 13 meeting (graciously hosted by BNY Mellon), where JCP Executive Committee members from major enterprises (bios below) will share their insights on adopting the new release and reveal their favorite features.

Join our lovable nerd party to learn about the new features in Java 21, and...let's celebrate the 25-year anniversary of the JCP!

Pizza and soft drinks will be served.

JCP Executive Committee Biographies

Chandra Guntur is a Distinguished Engineer at BNY Mellon. Chandra is a technologist in the financial services industry since 2003 and is programming with Java since 1998. Chandra was selected as a 2019 Java Champion. He is a Java User Group (JUG) Leader and helps run one of the largest Java user groups, NYJavaSIG and a founder-leader at the Garden State JUG.

Rodrigo Graciano is a Director and Senior Principal Software Engineer at BNY Mellon. He has more than 15 years of experience as Java Developer. Rodrigo is a Java Champion and a leader of the NYJavaSIG. He is also a frequent speaker at conferences including Java One, Devnexus, KCDC, TDC, and Java User Groups.

Alex Belokrylov is the CEO and co-founder of BellSoft. Alex's Java journey started at Sun Microsystems and was followed by his Oracle experience, which helped to establish Alex as one of the most influential Java thinkers and a significant contributor to Java's evolution and enterprise adoption.

Brian Stafford is the Principal Software Engineering Manager of the Java Virtual Machine team at Microsoft. His team helps to maintain and extend OpenJDK, working on such things as improvements to Escape Analysis / Scalar Replacement and reimplementing Java Object Monitors in Java.

Ken Fogel is a professor in the Computer Science Technology program at Dawson College in Montreal. In 2019 Ken became Java Champion, and he also began a free one-day conference in Montreal called DawsCon. He has attracted some of the finest presenters to speak to both students and the local developer community.

Bruno Caballero works for MicroDoc Computersysteme GmbH and is based in Berlin, Germany. During his career, Bruno has developed a wide range of embedded software projects. Today he is involved in porting the Java virtual machine to new hardware platforms and operating systems, as well as supporting customers on creating Java solutions for embedded devices.

Bruno Souza is a Java Developer and founder of SouJava (The Brazilian Java Technology Users Society). Known as the Brazilian Javaman, Bruno Souza helps professional Java developers to grow their technical careers. Since 1995 working with Java, Bruno is the president of SouJava, and represents the JUG in the JCP EC.

Dan Heidinga is a Senior Principal Software Engineer working for Red Hat on OpenJDK. Dan is a member of the Project Valhalla Expert Group, a committer to Project Leyden and Project CRaC, and dabbles in other parts of the runtime. He's also an Eclipse OpenJ9 Project Lead.

Yishai Galatzer is a Director of Software Engineering at Amazon Web Services (AWS). In this role, Yishai leads teams working on languages and runtimes at AWS, including OpenJDK, Amazon Corretto, and the Rust programming language, along with support for the use of JVM and Rust across Amazon. As part of AWS's work with OpenJDK, Yishai's team makes upstream contributions focused on long-term support, security, performance, observability, and availability.
